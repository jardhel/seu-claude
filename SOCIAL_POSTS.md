Hey HN!

I built seu-claude to solve the "goldfish memory" problem with Claude Code's memory plugins.

The problem: Most RAG solutions for coding assistants only remember files the AI has touched, use in-memory vector DBs (35GB+ RAM), and require Python with CUDA dependencies.

My solution:
- Proactive indexing of your entire codebase
- AST-based semantic chunking using Tree-sitter (not naive text splitting)
- LanceDB for disk-based vector storage (~100MB RAM idle)
- Transformers.js for 100% local embeddings
- Pure Node.js - no Python dependencies

It works as an MCP server for Claude Code and Claude Desktop.

Install: npm i -g seu-claude

Tech stack: TypeScript, Tree-sitter (WASM), LanceDB, Transformers.js (Xenova/all-MiniLM-L6-v2)

GitHub: https://github.com/jardhel/seu-claude

Would love feedback, especially from developers who've struggled with memory/context in AI coding assistants!
